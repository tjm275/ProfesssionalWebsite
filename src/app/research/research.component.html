At Cornell University, I worked in the Software Engineering Lab under Owolabi Legunsen. In this lab I worked in a field called runtime verification, which can be summarized succinctly as a birdge between formal methods of verifying software and normal testing. Runtime verification works by checking program executions against formal mathmatical specifications. One challenge with runtime verification is the writing of said specifications, which are required for RV to work. These need to be written carefully in formal languages, so it is not always possible for developers to spare the time to write them.

One development in the field are a class of tools known as "specification miners". These tools look at program executions and infer specifications. The idea is that these executions model correct behaviour, so if we enforce correct behaviour, we can avoid bugs. However, many of these mined specifications are trivial, or enforce behaviour that is not strictly required for program correctness. To remedy this issue Temari, perturbs a program such that it violates a specification. If the program behaves correctly while violating the spec, the spec is clearly not required for correctness, and it can be thrown out.
